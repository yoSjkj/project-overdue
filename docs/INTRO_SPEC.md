# 인트로 구현 지시서 (INTRO_SPEC.md)

이 문서는 게임 "(제목 미정)"의 인트로 시퀀스를 구현하기 위한 상세 명세서입니다.
Claude Code가 이 문서를 보고 바로 구현할 수 있도록 화면 상태, 전환 조건, 대사, 연출을 구체적으로 정의합니다.

---

## 전체 흐름 요약

```
[타이틀] → [인트로: 일상 서점] → [전환 연출] → [이면세계 진입] → [첫 전투 (튜토리얼)]
```

총 목표 시간: 타이틀 → 첫 전투 시작까지 약 30~40초 (플레이어 탭 속도에 따라 다름)

## Screen 2: 일상 서점 (인트로)

### 레이아웃 (모바일 세로 기준)

```
┌─────────────────────┐
│                     │
│    [서점 배경]       │
│    책장 + 따뜻한 조명 │
│                     │
│    [주인공]          │
│    뒷모습, 책장 앞    │
│                     │
│                     │
│  [대사 영역]         │
│  화면 하단 반투명 박스 │
│                     │
│  [버튼: 책 정리하기]  │
└─────────────────────┘
```

### 상태 머신

이 화면은 `introStep` 상태값으로 관리. 버튼 탭마다 다음 스텝으로 진행.

```
introStep: 0 → 1 → 2 → 3 → 4 (전환 트리거)
```

### Step별 상세

#### Step 0: 초기 상태

- 서점 배경 표시 (따뜻한 색감: 베이지/아이보리 톤)
- 주인공 뒷모습 표시 (책장 앞에 서 있음)
- 주인공 idle 애니메이션: 미세하게 위아래 움직임 (CSS transform translateY, 2px, 2초 주기)
- 대사: 없음
- 버튼: "책 정리하기"

#### Step 1: 첫 번째 탭

- 주인공 책 정리 모션 (스프라이트 교체 또는 간단한 팔 움직임 애니메이션)
- 모션 없으면: 주인공 위치 살짝 이동 후 복귀 (0.3초)
- 대사 표시: **"책이 뒤 섞여있네..."**
- 대사 표시 방식: 한 글자씩 타이핑 효과 (40ms/글자)
- 버튼: "책 정리하기" (유지)

#### Step 2: 두 번째 탭

- 주인공 책 정리 모션 반복
- 대사 표시: **"손님은 없고 책만 많고..."**
- 버튼: "책 정리하기" (유지)

#### Step 3: 세 번째 탭 — 이상한 책 발견

- 주인공 모션: 멈춤 → 고개 갸웃 (스프라이트 교체 또는 살짝 기울임)
- 화면 연출: 책장 한 곳에서 미세한 빛 반짝임 (CSS glow 또는 작은 빛 파티클)
- 대사 표시: **"이 책은 뭐지...?"**
- 버튼 텍스트 변경: **"꺼내보기"** (버튼 색상 또는 스타일도 미묘하게 변화 — 약간 붉거나 금빛)

#### Step 4: "꺼내보기" 탭 → 전환 연출 트리거

- 버튼 비활성화 (연타 방지)
- Screen 3 (전환 연출)으로 진행

### 대사 표시 규칙

- 위치: 화면 하단, 반투명 검정 박스 (opacity 0.7)
- 폰트: 게임 기본 폰트, 흰색
- 타이핑 효과: 한 글자씩 나타남 (40ms 간격)
- 대사 완료 후 버튼 활성화 (타이핑 중에는 버튼 탭해도 대사 스킵 → 전체 표시)

## Screen 3: 전환 연출

### 타임라인 (자동 진행, 플레이어 입력 없음)

```
0.0초  주인공 손 뻗는 모션 (또는 주인공 위치가 책장 쪽으로 살짝 이동)
0.3초  책에서 빛 발산 시작 (화면 중앙에서 원형 glow 확대)
       - CSS: box-shadow 또는 radial-gradient 애니메이션
       - 색상: 금색 → 보라색으로 변화
0.5초  화면 흔들림 시작 (screen shake)
       - 컨테이너 div에 transform translate 랜덤 적용
       - 강도: ±3px → 점점 ±8px로 증가
       - 주기: requestAnimationFrame 또는 setInterval 50ms
0.8초  글리치 효과 시작
       - RGB 분리: 배경/캐릭터 레이어를 R/G/B 별로 2~3px 오프셋
       - 또는 간단하게: CSS filter로 hue-rotate 빠르게 변화
       - 노이즈 오버레이: 반투명 랜덤 픽셀 레이어 (Canvas 또는 CSS background)
1.0초  백색 플래시
       - 화면 전체 흰색 오버레이, opacity 0→1 (0.1초)
1.1초  배경 교체
       - 서점 배경 → 이면세계 서점 배경으로 교체
       - 이면세계 색감: 어두운 보라/남색 톤, 같은 구도
       - 교체 후 흰색 오버레이 opacity 1→0 (0.2초)
1.3초  흔들림 서서히 멈춤 (강도 ±8px → 0px, 0.3초에 걸쳐)
1.5초  주인공 대사: **"...어...? 여기가... 어디...?"**
       - 타이핑 효과 적용
2.0초  대사 완료 후 0.5초 대기
2.5초  Screen 4로 자동 전환 (적 등장 → 첫 전투)
```

### 구현 노트

- 전체 시퀀스를 하나의 async 함수 또는 setTimeout 체인으로 관리
- 각 단계는 Promise나 setTimeout으로 타이밍 제어
- 글리치 효과가 구현 어려우면 최소한 **화면 흔들림 + 백색 플래시 + 배경 교체**만으로도 OK
- 우선순위: 백색 플래시 > 화면 흔들림 > 글리치 (복잡한 순서대로 후순위)

### 이면세계 배경

- 일상 서점과 동일한 레이아웃/구도
- 색감만 변경: 따뜻한 베이지 → 어둡고 차가운 보라/남색
- v1에서는 CSS filter로 기존 배경 색상 변경해도 OK:
  ```css
  .otherworld {
    filter: hue-rotate(200deg) brightness(0.4) saturate(1.5);
  }
  ```
- 나중에 별도 배경 이미지로 교체 가능

## Screen 4: 첫 전투 (튜토리얼)

### 진입 조건

- Screen 3 전환 연출 완료 후 자동 진입

### 적 등장 연출

```
0.0초  화면 중앙 상단에 적 등장 (페이드 인 또는 아래에서 슬라이드 업, 0.3초)
       - 첫 적: "촉수 북마크" (약한 적)
       - 플레이스홀더: 색깔 네모 + 적 이름 텍스트
0.3초  책(안내서) 등장
       - 화면 하단에 펼쳐진 책 아이콘/이미지 등장 (페이드 인)
       - 또는 대사 영역 옆에 책 아이콘 표시
0.5초  책 대사 시작 (타이핑 효과)
```

### 전투 튜토리얼 — 대사 시퀀스 (상태 머신)

```
tutorialStep: 0 → 1 → 2 → 3 → 4 → 5 → 6 → 7 → 튜토리얼 종료
```

#### tutorialStep 0: 책 자기소개

- 책 대사: **"안녕하세요. 저는 안내서입니다. 열람해주셔서 감사합니다."**
- 자동 진행 (대사 완료 후 1초 대기 → 다음)

#### tutorialStep 1: 주인공 반응

- 주인공 대사: **"...책이 말을 한다고...?"**
- 자동 진행 (1초 대기)

#### tutorialStep 2: 책 설명

- 책 대사: **"네, 물론이죠. 책이 말을 안 하면 누가 하나요?"**
- 자동 진행 (1초 대기)

#### tutorialStep 3: 적 안내

- 책 대사: **"앞에 연체자가 있네요. 제 페이지를 사용하시면 처리할 수 있습니다."**
- 이 대사 완료 후 → 손패에 카드 3장 표시 (기본 공격 2장 + 기본 방어 1장)
- 카드 등장 애니메이션: 화면 하단에서 슬라이드 업 (0.3초)

#### tutorialStep 4: 카드 사용 안내

- 공격 카드에 펄스 애니메이션 (CSS: scale 1.0 ↔ 1.05, 반복)
- 책 대사: **"밝게 빛나는 페이지가 보이시죠? 가볍게 터치해주시면 됩니다."**
- 플레이어 입력 대기: 공격 카드 탭

#### tutorialStep 5: QTE 안내

- 카드 탭 → QTE 화면 표시
  - QTE UI: 원형 게이지 또는 타이밍 바 (구현 편한 것으로)
  - 타이밍 윈도우: **매우 관대 (±500ms)** — 거의 무조건 성공하게
- 책 대사: **"문자가 나타나면 적절한 타이밍에 터치해주세요."**
- 플레이어 입력 대기: QTE 탭

#### tutorialStep 6: QTE 결과

- 성공 시 → 책 대사: **"훌륭하십니다."**
- 실패 시 → 책 대사: **"괜찮습니다. 오역은 누구나 하는 거니까요."**
- 적에게 데미지 적용 + 데미지 숫자 표시
- 주인공 대사: **"어...? 됐어...?"**
- 자동 진행 (1초 대기)

#### tutorialStep 7: 적 턴 + 방어 안내

- 적의 공격 인텐트 아이콘 표시 (검 아이콘 + 데미지 숫자)
- 적이 공격 → 주인공 HP 감소 → 주인공 대사: **"아파...!!"**
- 다음 턴 시작 → 방어 카드에 펄스 애니메이션
- 책 대사: **"다음 공격이 예상됩니다. 방어 페이지 사용을 권장드립니다."**
- 플레이어 입력 대기: 방어 카드 탭

#### 튜토리얼 종료 후

- 이후 턴은 안내 없이 자유 플레이
- 적 처치 시:
  - 책 대사: **"연체 도서가 회수되었습니다. 이용해주셔서 감사합니다."**
  - 주인공 대사: **"뭐가 어떻게 된 거야..."**
- 카드 보상 화면으로 전환 (3장 중 1장 선택)

### 첫 전투 특수 규칙

| 항목 | 첫 전투 | 2번째 전투부터 |
|------|---------|--------------|
| 카드 수 | 3장 (공격2 + 방어1) | 5장 (정상) |
| 잉크(마나) | 미적용 (전부 사용 가능) | 3/턴 (정상) |
| QTE 윈도우 | ±500ms (매우 관대) | ±300ms/±100ms (정상) |
| 안내 대사 | 있음 (tutorialStep) | 없음 |
| 적 강도 | 최약 (HP 15, 공격 3) | 정상 (GDD 밸런싱 기준) |

## 대사 표시 시스템 공통 규칙

### 화자 구분

```
책(안내서): 대사 앞에 책 아이콘 표시. 텍스트 색상: 금색 또는 연한 베이지
주인공:     대사 앞에 주인공 아이콘 표시. 텍스트 색상: 흰색
```

### 타이핑 효과

- 속도: 40ms/글자 (기본)
- "..." 일 때: 각 점마다 200ms 대기 (극적 효과)
- 탭하면 타이핑 스킵 → 전체 텍스트 즉시 표시

### 대사 영역

- 위치: 화면 하단
- 스타일: 반투명 어두운 박스 (padding 16px, border-radius 8px)
- 화자 아이콘: 왼쪽 32x32
- 텍스트: 아이콘 오른쪽

## 상태 관리 구조 (Zustand 권장)

```javascript
// introStore.js
const useIntroStore = create((set) => ({
  // 현재 게임 페이즈
  phase: 'title', // 'title' | 'intro' | 'transition' | 'battle'
  
  // 인트로 진행 단계
  introStep: 0, // 0~4
  
  // 튜토리얼 진행 단계
  tutorialStep: 0, // 0~7
  
  // 현재 대사
  currentDialogue: null, // { speaker: 'book'|'player', text: '...' }
  
  // 전환 연출 진행 중 여부
  isTransitioning: false,
  
  // Actions
  nextIntroStep: () => set((state) => ({ introStep: state.introStep + 1 })),
  setPhase: (phase) => set({ phase }),
  setDialogue: (dialogue) => set({ currentDialogue: dialogue }),
  setTransitioning: (v) => set({ isTransitioning: v }),
}));
```

## 구현 우선순위

반드시 이 순서로 구현:

```
1순위: 타이틀 → 인트로 화면 전환 (기본 상태 머신)
2순위: 인트로 Step 0~4 (버튼 탭 → 대사 표시 → 버튼 텍스트 변경)
3순위: 전환 연출 (최소: 백색 플래시 + 배경색 변경)
4순위: 첫 전투 진입 + 책 대사 시퀀스
5순위: 전환 연출 강화 (화면 흔들림, 글리치)
```

1~3순위까지만 되면 인트로는 플레이 가능한 상태.
